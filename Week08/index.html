Pattern, Rhythm, & Movement

Concepts

I tried to create multiple different sketches that allowed me to explore what I could do with p5.js.

For sketch 01, I utilized an example we created in class in order to explore blend modes and how to translate/rotate a shape within the grid. For sketch 02, I tried to create a more complex pattern that used multiple shapes, arcs, and random colors. Finally, for sketch 03, I wanted to create a pattern that utilized movement within the grid, ableit the movement being linear and simple.

Problems & Solutions

  1. Working with HSB:: This color mode was a bit difficult for me to use just because I didn't intuitively know what number would point to which hue. I mostly used this as an opportunity to explore the color mode more and experiment but I also played around in Adobe Illustrator's color picker to see how all the numbers correlated with each other.

  2. Rotating & translating shapes: Aside from figuring out the math behind the rotation of the squares in sketch 01, this task was pretty easy. However, I didn't realize that translations and rotations accumulated every time you called the function. I had to do some Googling and looking around the references to discover that push() and pop() would help restore the original state pre-transformation and prevent the accumulation of previous transformations.

  3. Drawing arcs: For sketch 02, I had the idea of creating a Bauhaus-esque pattern since I felt that this covered topics we discussed in class while also presenting the challenge of creating a grid within a grid and drawing arcs. The math behind the grid within the grid was simple enough since it mostly just involved adding the shape's size to whatever coordinate it was applicable to. However, drawing the arcs was more of a struggle due to me not fully understanding the math behind it.

    I solved this by reviewing the resources on drawing arcs. The first four parameters of where to place the arc and what size it needed to be were straightforward since we had gone over it in class, but the additional resources were helpful regarding the fifth and sixth parameters for the start and stop points. I ended up looking up a radian circle that showed me the points in which I needed to place my start and stop points. I then had to look up what the parameters were for radians since it looked like p5.js had preset ones like PI and HALF_PI that didn't involve me typing in a mathematical formula to calculate the specific angle I wanted in radians.

  4. Combining everything we learned & implementing movement:: For my third sketch, I wanted to use the grid system, rotation, random color generation, and scale but as an animated pattern instead. I created a static background that randomly generated colors in each box. On top of that, I created another grid of a different size that implemented movement. I used what I learned about push() and pop() from working on my first sketch to help me create the repeating animation in each grid segment. My main problem with this sketch was mostly trying to implement the gradient effect of both the rotating line and the growing circle. This sketch was primarily experimental for me so I messed around a lot with gradually incrementing different variables in my code such as the radius, saturation, and turn.
